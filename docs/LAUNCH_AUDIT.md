### Canonical-direction note

`<link rel="canonical">` points at `https://chekkify.com` (apex) but the apex 307-redirects to `https://www.chekkify.com`. Google generally tolerates this, but the redirect target and the canonical target should ideally agree to avoid signal split. Consider switching the canonical to `https://www.chekkify.com` (or making `chekkify.com` the live origin and redirecting `www` -> apex). Low priority.

## Critical Issues

None that block launch. Items worth fixing **soon after launch**:

1. **Missing `hreflang` tags** — TR/EN content exists but isn't signalled to search engines. Add `<link rel="alternate" hreflang="tr" ...>` / `hreflang="en"` / `hreflang="x-default"` in the root layout `metadata.alternates.languages`.
2. **Canonical points to non-200 origin** — see note above.
3. **Frontend security headers thin** — no `Referrer-Policy`, no `Permissions-Policy` on Vercel responses. Add in `next.config.ts` `headers()` or `middleware.ts`.
4. **No error tracking wired up** — no `@sentry/nextjs` / `@sentry/node` in either `dashboard/package.json` or `backend/package.json`. Production errors are currently only visible via Vercel + Railway logs.
5. **No uptime monitoring** — `api.chekkify.com/health` returns 200 JSON but nothing watches it.

## Recommendations

- **Performance** — once a real Lighthouse run is available, focus on: image lazy-loading (`next/image` `loading="lazy"`), font subsetting/`font-display: swap`, JS chunk inspection (`@next/bundle-analyzer`), and removing unused locales from the i18n shipment.
- **SEO** — add `hreflang`, align canonical with the live origin, submit sitemap to Google Search Console + Bing Webmaster Tools.
- **Security headers** — add `Referrer-Policy: strict-origin-when-cross-origin` and a minimal `Permissions-Policy` on the frontend.
- **Observability** — wire Sentry (frontend + backend) and a Better Stack uptime monitor before promoting heavy paid traffic.
- **Schema.org** — current `SoftwareApplication` + `AggregateOffer` is good; consider adding `BreadcrumbList` per page once the blog grows.
